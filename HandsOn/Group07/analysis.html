Data and Ontology Analysis

Dataset: Air Quality in Madrid (2024)

This document presents the analysis of the dataset used for the Linked Data generation process. It includes the data structure inspection, licensing evaluation, and the proposed resource naming strategy that will be applied to the ontology and RDF data. The dataset originates from the Madrid City Council Open Data Portal and contains daily measurements of air quality indicators such as NO₂, PM₁₀, O₃, SO₂, and CO collected across different monitoring stations in Madrid during 2024.

1. Dataset Structure Analysis

The dataset is provided in CSV format and includes one record per pollutant and station per day. Below is a summary of the main columns and their characteristics:

Column Description Example Value Data Type Notes
ESTACION Code identifying the air quality monitoring station. 4 Integer Links to real-world stations; can be connected to Wikidata or OSM.
PUNTO_MUESTREO Station code including the district and instrument details. 28079004 String Acts as a unique identifier for a specific measurement point.
MAGNITUD Pollutant identifier (e.g., 8 = NO₂, 9 = O₃). 8 Integer Defines the type of pollutant measured.
FECHA Date of the measurement. 2024-05-03 Date Used to build time-series data and temporal queries.
VALOR Average daily value of the pollutant in µg/m³. 45 Float Primary measurement value.
UNIDAD Unit of measurement (typically µg/m³). µg/m³ String Could be linked to an ontology of measurement units (e.g., OM).

The dataset is clean and mostly structured, with very few missing values. It is suitable for RDF transformation and can easily be enriched with contextual data such as station location or meteorological information.

2. Licensing Analysis

The dataset is published under the Madrid City Council Open Data License, which allows reuse for both commercial and non-commercial purposes as long as attribution is provided. It aligns closely with the Creative Commons Attribution 4.0 (CC BY 4.0) license.

  • Rightsholder: Ayuntamiento de Madrid (Madrid City Council)
  • Access URL: https://datos.madrid.es/
  • License terms: Reuse permitted with mandatory attribution.
  • Proposed license for generated dataset: CC BY 4.0 International

The generated Linked Data will include attribution metadata, maintaining license compatibility with other open datasets.

3. Resource Naming Strategy

The URIs for the ontology and dataset will follow the recommendations provided in the Methodological Guidelines for the Generation of Linked Data. The naming scheme uses both hash and slash URIs depending on the resource type.

  • Domain: http://smartcity.linkeddata.es/group07/
  • Ontology path: ontology/AirQuality# (for classes and properties)
  • Resource path: resource/ (for individuals)
  • Example ontology term: http://smartcity.linkeddata.es/group07/ontology/AirQuality#AirQualityObservation
  • Example individual: http://smartcity.linkeddata.es/group07/resource/Observation/2024-05-03-04

This structure separates the ontology model from its instances and ensures persistence, readability, and compliance with Linked Data best practices. The ontology itself will reuse existing vocabularies such as schema:Place and ssn:Observation to increase interoperability.

4. Conclusions

The Air Quality in Madrid (2024) dataset meets all requirements for Linked Data generation. It will be transformed into RDF using the defined ontology, which models air quality observations, pollutants, and stations. The resource naming strategy guarantees URI persistence and interoperability with other smart city datasets.
